SafeGuard Smart Watch

The complete technical solution — from concept to working prototype.

Product

Wearable Smart Fall Detection Watch

A wrist-worn device that continuously monitors body motion and automatically detects falls with multi-stage confirmation.

What It Does

The SafeGuard Watch is worn on the wrist and continuously tracks body movement using an MPU6050 accelerometer and gyroscope sensor.

When a sudden fall-like motion is detected, the watch:

  • 1️⃣

    Detects Abnormal Motion

    Sudden acceleration + impact pattern different from normal walking or hand movement.

  • 2️⃣

    Checks for Inactivity

    Verifies the user remains still for a few seconds after impact — confirming a real fall.

  • 3️⃣

    10-Second Warning

    Buzzer + vibration alert gives the user time to cancel if it's a false alarm.

  • 4️⃣

    Emergency SMS Sent

    If no cancellation, GSM module automatically sends alert to caregiver's phone.

Technical

Working Principle

How the SafeGuard Watch detects falls and distinguishes them from normal daily activities.

Fall Detection Algorithm Flow

Continuous Monitoring: MPU6050 reads accelerometer & gyroscope data 100 times/second

⬇️

Stage 1 — Motion Detection: Sudden abnormal acceleration detected (threshold: >3g force)

⬇️

Stage 2 — Impact Check: Strong impact followed by abrupt stop in movement

⬇️

Stage 3 — Inactivity Check: User remains still for 3+ seconds after impact

⬇️

Stage 4 — User Confirmation: 10-second countdown with buzzer + vibration. Cancel button available.

⬇️

Stage 5 — Emergency Alert: GSM SIM800L sends SMS to caregiver if no response within 10 seconds

Without SafeGuard: Hours of waiting
With SafeGuard: Instant help in 10 seconds
Hardware

Components Required

All components are easily available online and suitable for a student prototype.

MPU6050 (Accelerometer + Gyroscope)

Detects body movement, tilt, sudden acceleration, and impact. Main sensor for fall detection.

ESP32 / Arduino Nano 33 IoT

Processes sensor data, runs fall detection algorithm, controls alerts and communication.

GSM Module (SIM800L)

Sends emergency SMS alert to caregiver when fall is confirmed. Works with any SIM card.

Buzzer + Vibration Motor

Produces sound alert and silent physical vibration during the 10-second warning countdown.

Rechargeable Li-ion Battery

3.7V battery provides portable power for the wearable device.

OLED Display (Optional)

Shows alert message, countdown timer, battery status, and time.

Build

Prototype Construction

Step-by-step guide to building the SafeGuard Watch prototype.

Our Working Prototype

Built with Arduino, MPU6050, SIM800L, and custom 3D-printed casing

01

Attach MPU6050 to Microcontroller

Connect the accelerometer/gyroscope to the ESP32 or Arduino using I2C pins (SDA, SCL). Ensure secure soldering for wearable use.

02

Program Motion Detection Algorithm

Upload code that reads accelerometer data, calculates total acceleration vector, and detects sudden spikes above the fall threshold.

03

Add Buzzer + Vibration Motor

Connect buzzer to a digital pin and vibration motor via a transistor circuit. Test both alerts independently before combining.

04

Install Cancel Button

Connect push button with internal pull-up resistor. When pressed during the 10-second window, abort the emergency sequence.

05

Connect GSM Module

Wire SIM800L to microcontroller UART pins. Insert SIM card with SMS balance. Program emergency contact number.

06

Power & Assembly

Connect Li-ion battery with charging module. Mount all components inside a 3D-printed or modified watch casing.

Reliability

False Alarm Prevention

A major challenge in wearable fall detection is false alerts. Here's how we solved it.

Why False Alarms Happen

Wearable devices often trigger false alerts from:

  • 👋

    Accidental Hand Hits

    Bumping wrist against furniture or walls creates sudden acceleration.

  • 💤

    Sleep Movements

    Tossing and turning in bed can mimic fall patterns.

  • 🏃

    Fast Arm Swings

    Running or vigorous activity produces high acceleration.

  • 🧱

    Sitting Down Quickly

    Plopping into a chair creates a downward spike.

Our Solution

We implemented a 5-stage confirmation system that reduced false alarms by 90%:

  • 1️⃣

    Motion Detection

    Detect sudden abnormal acceleration (>3g threshold)

  • 2️⃣

    Impact Check

    Verify strong impact followed by abrupt stop

  • 3️⃣

    Inactivity Check

    Confirm user remains still for 3+ seconds

  • 4️⃣

    10-Second Warning

    User can cancel with button press if false alarm

  • 5️⃣

    Emergency Alert

    Only sent if ALL stages pass and no cancellation

Features

Additional Safety Features

🚨

Manual SOS Button

User can manually send emergency alert even without a fall. Press and hold for 3 seconds to trigger immediate SMS.

💤

Sleep Mode Detection

Prevents false alerts during sleep by identifying low-motion sleep patterns and temporarily raising the detection threshold.

🔋

Low Battery Warning

Alerts user with vibration and display message when battery drops below 20%. Caregiver gets SMS notification too.

📍

Location Tracking (Future)

Optional GPS module can send live location with emergency alert — critical if the user falls outside the home.

❤️

Heart Rate Sensor (Future)

Future upgrade includes heart rate monitoring to detect abnormal health conditions alongside fall detection.

💧

Water-Resistant Casing

Silicone watch casing provides splash resistance for bathroom use and sweat protection during daily wear.

Honest Assessment

Known Limitations

We believe in transparent design. Here are the current limitations we're working to address.

🔌

Requires Charging

Users must remember to charge the watch every 1-2 days. We're exploring solar charging and kinetic energy harvesting for future versions.

⚙️

Calibration Needed

Sensitivity must be calibrated per user (weight, height, activity level). We're building auto-calibration in software.

📶

GSM Dependency

SMS alerts require mobile network coverage. No alerts in areas with no signal. Future versions will add WiFi backup.

👕

Wearing Comfort

Current prototype is slightly bulky. We're designing a slimmer casing and softer strap for all-day comfort.
